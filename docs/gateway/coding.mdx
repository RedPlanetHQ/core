## Overview

The Coding capability lets CORE spawn coding agents like Claude Code on your machine. Start a coding session from WhatsApp, check its progress from the web dashboard, and let it work autonomously while you're away from your computer.

Sessions run in the background and persist their output, so you can resume or read results anytime.

---

## Supported Agents

Currently supported:

- **Claude Code** - Anthropic's CLI coding agent

More agents coming soon. The system is extensible to support any CLI-based coding agent.

## Starting sessions from WhatsApp

Use this when you want to kick off a Claude Code session without opening your laptop. You send one message, CORE starts a gateway coding session on your machine, and then keeps you posted until it’s done.

### Recommended message templates

**Template A (fastest, recommended)**

> start a claude code session
> repo: ~/Github/core
> branch: fix/auth-timeout
> task: fix the auth timeout bug in the api, add tests, and open a PR

**Template B (with issue link + commands)**

> start coding
> repo: ~/Github/core
> branch: fix/658-whatsapp-sessions-docs
> issue: https://github.com/RedPlanetHQ/core/issues/658
> task: update docs/gateway/coding.mdx with a “starting sessions from WhatsApp” section
> commands: pnpm lint, pnpm test

**Template C (specify model)**

> start a claude code session
> repo: ~/Github/core
> branch: feat/xyz
> model: claude-3-7-sonnet
> task: implement xyz and keep changes minimal

### Required fields

- **repo** (repo path / working dir)
  - Example: `~/Github/core` or `/Users/manik/Github/core`
- **branch** (name to create + checkout)
  - Example: `fix/auth-timeout`
- **task** (what you want done)
  - Example: “fix login redirect loop; add tests; open a PR”

### Optional fields

- **issue** (GitHub issue URL)
  - Helps CORE pull exact context and link the work back
- **commands** (what you want run)
  - Example: `pnpm lint`, `pnpm test`, `pnpm typecheck`
- **model** (if you care)
  - Otherwise CORE uses the default configured model

### If you forget something, CORE will ask

If your message doesn’t include the required info, CORE will reply with a short set of prompts, typically:

- “what repo path should i use?”
- “what branch name should i create + checkout?”
- “what’s the exact task / outcome?”
- “any commands you want me to run before i send the final summary?”

### Git workflow note (what CORE will do)

To keep changes clean and PR-friendly, CORE will typically do:

1. checkout the default branch (usually `main`)
2. pull the latest
3. create and checkout your requested branch

If you want something different (e.g., branch from `develop`), include it explicitly.

### What “keep session running” means

Sometimes CORE will say it will “keep the session running.” That means:

- the gateway process continues running the coding agent in the background
- you can come back later and ask for status or output without restarting the work

It does **not** mean the agent will run forever — it will stop when the task is complete (or when you ask CORE to close it).

### Progress + status updates

CORE reports progress in milestones so you can sanity-check without reading a huge log. Expect updates like:

- **started**: session created + agent launched
- **plan**: a short plan + what it’s going to do next
- **checkpoint(s)**: “tests passing”, “fix landed”, “docs updated”, “PR ready”, etc.
- **needs input** (only if blocked): missing credentials, unclear requirement, failing tests with ambiguity

To check output at any time, ask something like:

- “status of the coding session?”
- “show last 50 lines of output”
- “summarize what changed so far”

When complete, CORE will send a completion summary that includes:

- what it changed (high level)
- where it changed it (files)
- commands run + results
- next action for you (e.g., “review PR”, “merge”, “deploy”)

## Commands

### Start a Session

```bash
corebrain coding start
```

This prompts you for:
- Agent to use (e.g., `claude-code`)
- Working directory
- Initial prompt

Or provide everything directly:

```bash
corebrain coding start --agent claude-code --dir /path/to/project --prompt "Fix the auth bug"
```

### List Sessions

```bash
corebrain coding list
```

Shows all sessions with their status (running, completed, closed).

### Read Session Output

```bash
corebrain coding read <sessionId>

# Read last 50 lines
corebrain coding read <sessionId> --lines 50 --tail
```

### Resume a Session

```bash
corebrain coding resume <sessionId>
```

Prompts for a new message to continue the session.

### Close a Session

```bash
corebrain coding close <sessionId>
```

Stops a running session.

## Gateway Tools

When the gateway is running, CORE can use these coding tools:

### coding_start_session

Start a new coding session:

```json
{
  "agent": "claude-code",
  "prompt": "Fix the authentication timeout issue",
  "dir": "/Users/me/projects/myapp"
}
```

Optional parameters:
- `model` - Specific model to use
- `systemPrompt` - Custom system prompt

### coding_resume_session

Resume an existing session:

```json
{
  "sessionId": "abc-123",
  "prompt": "Now add tests for the fix"
}
```

### coding_read_session

Read session output:

```json
{
  "sessionId": "abc-123",
  "lines": 100,
  "tail": true
}
```

### coding_close_session

Close a session:

```json
{
  "sessionId": "abc-123"
}
```

### coding_list_sessions

List all sessions (no parameters).

## Session Lifecycle

<Steps>
  <Step title="Start">
    CORE starts a session with an agent, directory, and prompt. The session gets a unique ID.
  </Step>

  <Step title="Running">
    The agent runs in the background. Output is streamed to a session file.
  </Step>

  <Step title="Monitor">
    CORE (or you) can read the session output at any time while it's running.
  </Step>

  <Step title="Complete">
    The agent finishes its task. Session status changes to "completed".
  </Step>

  <Step title="Resume or Close">
    Resume with a new prompt to continue, or close to clean up.
  </Step>
</Steps>

## Configuration

Configure agents before first use:

```bash
corebrain coding config --agent claude-code
```

This sets up the command path and any agent-specific options.

## Use Cases

**Remote bug fixing:** You're at dinner, a P0 bug comes in. Message CORE: "Fix the payment processing error in checkout.ts". CORE spawns Claude Code, provides context from memory, and works on the fix.

**Async code reviews:** "Review the changes in the auth PR and suggest improvements." CORE starts a session, analyzes the code, and sends you a summary when done.

**Automated refactoring:** "Migrate all API routes from Express to Hono." CORE works through the codebase file by file, and you check in periodically to see progress.

**Night shifts:** Set a reminder: "At 2am, run the database migration script and report any errors." CORE handles it while you sleep.

---

## Notes

- Sessions are stored in `~/.corebrain/sessions.json`
- Session output is read from the agent's native session files
- Multiple sessions can run concurrently
- Sessions persist across gateway restarts
