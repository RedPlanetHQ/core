The CORE CLI (`corebrain`) connects to CORE Cloud by default. When running a self-hosted instance, you can point it to your own deployment instead.

## Connecting to Your Instance

When you run `corebrain login`, it will ask for a CORE instance URL. Pass your self-hosted instance URL:

```bash
corebrain login
# When prompted for instance URL, enter:
# https://your-domain.com
```

This connects the Gateway to your own CORE instance instead of CORE Cloud. Make sure `APP_ORIGIN` and `LOGIN_ORIGIN` are set correctly in your `.env` file: see [environment variables](/self-hosting/environment-variables).

## Setting Up a Gateway

Once you're logged into your self-hosted instance, run:

```bash
corebrain gateway setup
```

The wizard asks where the gateway should run (native / docker / railway) and handles config, security key generation, optional Tailscale funnel, and registration with your instance.

For the full reference — flags, the manifest API, capability slots, environment variables — see the [Gateway overview](/gateway/overview#install).
