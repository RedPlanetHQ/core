## Overview

Send emails to **brain@getcore.me** and get intelligent responses from CORE's Meta Agent. Perfect for longer requests, asynchronous workflows, or when you're already in your email client.

## Getting Started

Simply email **brain@getcore.me** from your registered email address. Your agent will:
1. Process your request
2. Search your memory if needed
3. Take actions across your connected apps
4. Reply with results and context

## Email Format

**Subject Line:**
Keep it descriptive but concise. The subject helps your agent understand the request type.

**Body:**
- Be specific about what you need
- Use bullet points for clarity
- Reference past context when relevant
- Specify which apps to interact with

**Attachments:**
- Supported formats: PDF, DOCX, TXT, images, spreadsheets
- Max size: 25MB per email
- Multiple attachments supported

## Example Emails

**Quick Action:**
```
To: brain@getcore.me
Subject: Create bug issue

Create a high-priority Linear issue:
"Login form validation not working on mobile Safari"
Assign to @sarah
```

**Memory Search:**
```
To: brain@getcore.me
Subject: Find API discussion

When did we discuss changing the webhooks API?
I need the full context and any decisions made.
```

**Multi-App Workflow:**
```
To: brain@getcore.me
Subject: Sprint planning prep

Help me prepare for sprint planning:
1. List all unassigned bugs in Linear
2. Get velocity from last 3 sprints
3. Find any GitHub issues labeled "technical-debt"
4. Create a summary document in Notion
```

## Response Time

- **Simple queries**: Usually within 1-2 minutes
- **Complex requests**: May take 5-10 minutes for thorough analysis
- **Batch operations**: Depends on number of actions (you'll get progress updates)

## Tips

- **Be specific**: Clear requests get better results
- **Use lists**: Bullet points help structure complex requests
- **Reference tools**: Mention which apps to use (Linear, Slack, etc.)
- **Include context**: Attach relevant files or reference past conversations

## Privacy & Security

- Only emails from your registered account are processed
- All communication encrypted in transit and at rest
- Attachments processed securely and not retained after analysis
- You can disable email access anytime in settings

## Limitations

- Must email from registered account address
- 25MB attachment limit per email
- Rate limited to prevent abuse (100 emails/day)
- Cannot send emails on your behalf (only receive and respond)
